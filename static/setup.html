Setup

We're planning on packaging CiteSeerExtractor so that no setup is required and so that It Just Works! However, until we do that, a little bit of work is needed to set it up. To setup CiteSeerExtractor you'll need to know a little bit about setting up Linux systems and installing packages.

Here we go...

Step 0: Dependencies

This isn't really a step in the setup but you'll need to at least satisfy these dependencies to get started:
  • Python >= 2.7
  • Java >= 1.6
There are other dependencies but we'll describe how to install them.

Step 1: Get the code

The code is publicly available on GitHub, so clone the git repository to get a copy of the code git clone git://github.com/SeerLabs/CiteSeerExtractor.git Easy, now comes the (kinda) hard part.

Step 2: Install the other dependencies

In addition to Python and Java, CiteSeerExtractor also depends on a number of packages. They are: web.py (for the Web server), xmltodict (a python package for converting XML to JSON), and String::Approx (a Perl module used during extraction). To install the packages issue the following commands.
  • sudo easy_install web.py (you can get easy_install from python-setuptools if you don't already have it)
  • sudo pip install xmltodict (you can get install pip with easy_install if you don't already have it)
  • sudo cpan String::Approx

Step 3: Make sure you have support for 32-bit executables

Some of the executables used in CiteSeerExtractor were compiled for 32-bit architectures. If you have a 32-bit machine, you can skip this step. Otherwise, you need to install support for 32-bit executables:
  • RHEL/CentOS: sudo yum install glibc.i686 libstdc++.i686
  • Ubuntu: sudo apt-get install ia32-libs-multiarch

Step 4: Run and test

Run the webserver with: python service.py [port] Navigate to http://localhost:port/extractor and try a document and check the console output for errors. Also, see how to use the API.
